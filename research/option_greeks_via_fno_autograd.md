With the advent of deep learning, surrogate models such as the FiLM-conditioned Fourier Neural Operator (FNO) have emerged to approximate complex mapping functions, such as the translation of Rough Heston parameters $\theta = (\kappa, \theta, \sigma, \rho, V_0, H)$ into an implied volatility surface $\sigma_{IV}(T,K)$. However, replacing closed-form analytical equations or traditional numerical differential equation solvers with neural surrogates introduces significant challenges in extracting exact risk sensitivities. The objective is not simply to predict the implied volatility, but to compute all Black-Scholes Greeks analytically via Automatic Differentiation (AD) using the PyTorch autograd engine, and subsequently convert these implied volatility Greeks back into model parameter sensitivities using the chain rule. This report provides an exhaustive, expert-level analysis of the mathematical foundations, computational architecture, portfolio applications, hedging simulations, and numerical validation protocols required to implement a production-grade FNO-to-Greek pipeline.Mathematical Foundations: Complete Taxonomy of Option GreeksThe bedrock of option risk management lies in quantifying the sensitivity of the derivative's price to various underlying market factors. This structural chain rule forms the fundamental basis upon which the PyTorch autograd engine executes its backward pass.PyTorch Autograd Mechanics for Exact Greek ExtractionThe extraction of derivatives from analytical formulas embedded in machine learning workflows has been revolutionized by Automatic Differentiation (AD). By reconstructing the closed-form Black-Scholes formulas utilizing differentiable PyTorch tensor operations, the framework guarantees exact calculations devoid of finite-difference truncation errors.Extracting First and Second-Order SensitivitiesTo compute primary Greeks, the initial variables, particularly the spot price $S$ and the implied volatility $\sigma_{IV}$, must be instantiated as tensors with the requires_grad=True flag. The torch.autograd.grad function allows one to execute a backward pass terminating at these specific nodes. To extract second-order Greeks, it is imperative to include the parameter create_graph=True during the first derivative extraction. This instructs PyTorch to construct a new computational graph dynamically during the backward pass, rendering the first derivative itself differentiable.Consequently, Gamma ($\Gamma = \frac{\partial^2 C}{\partial S^2}$) is computed by invoking torch.autograd.grad on the previously computed Delta tensor with respect to the spot price tensor. This procedure eliminates the need to explicitly code the complex closed-form Hessian matrices, relying instead on the exactness of dual-number arithmetic inherent in the autograd engine.Full Jacobian Computation: Forward-Mode vs. Reverse-Mode ADWhen projecting these sensitivities back to the 6 fundamental Rough Heston parameters, one encounters a mapping from a low-dimensional input space ($\mathbb{R}^6$) to a high-dimensional output space. A standard volatility surface grid may consist of 8 maturities and 11 strikes, yielding a function mapping $f: \mathbb{R}^6 \to \mathbb{R}^{88}$. The objective is to compute the full $88 \times 6$ Jacobian matrix.PyTorch 2.0 and the torch.func library (formerly functorch) expose two distinct Application Programming Interfaces (APIs) for Jacobian computation: jacfwd (forward-mode AD) and jacrev (reverse-mode AD). The choice of algorithm is strictly dictated by the dimensionality of the mapping. Reverse-mode AD (implemented via Vector-Jacobian Products, or VJPs) computes the Jacobian row-by-row. For an 88-dimensional output, jacrev executes 88 separate backward passes through the highly parameterized FNO network, sequentially constructing and dismantling massive computational graphs.Conversely, forward-mode AD (jacfwd), driven by Jacobian-Vector Products (JVPs), propagates primal and tangent values simultaneously from input to output. It computes the Jacobian column-by-column. Since there are only 6 inputs, jacfwd requires only 6 forward passes. Consequently, the PyTorch documentation and performance benchmarks indicate that jacfwd provides an order-of-magnitude performance acceleration over jacrev for neural operators mapping few parameters to dense spatial grids. Furthermore, jacfwd vastly outperforms classical finite difference approximations, which would require at least 12 forward passes (central difference) while suffering from inevitable numerical truncation and round-off errors.Vectorization and Memory ConstraintsIn institutional risk management, portfolio analysis requires the evaluation of Greeks across tens of thousands of unique contracts simultaneously. PyTorch provides torch.func.vmap, a vectorizing transform that converts single-sample operations into batched operations executed concurrently across GPU streaming multiprocessors without explicit for loops. The composite function vmap(jacfwd(fno_model)) computes the Jacobian for an entire batch of parameters in parallel.However, extracting second-order parameter sensitivities—the FNO Hessian $\frac{\partial^2 \sigma_{IV}}{\partial \theta_i \partial \theta_j}$ via torch.func.hessian—imposes severe memory constraints. Because the differential operators governing the Greeks are strictly linear, the aggregate risk of a portfolio containing $N$ options is the exact sum of the volume-weighted individual sensitivities. The formulation is expressed as:$$\text{Greek}_{port} = \sum_{i=1}^N w_i \times \text{Greek}_i$$where the weight $w_i$ represents the number of contracts held multiplied by the underlying contract multiplier (e.g., 100 shares for standard US equity options).Dollar Delta and Futures HedgingTo immunize an index options portfolio against directional asset movements, traders utilize Dollar Delta, defined as $\Delta_{\$} = \Delta_{port} \times S$. This metric represents the absolute fiat currency exposure to a 100% aggregate move in the underlying asset. To hedge this exposure, practitioners trade linear derivatives, predominantly equity index futures (e.g., SPX futures). If the notional value of a single future is denoted by $F_{notional}$ (the futures price multiplied by its point value), the requisite number of futures contracts to buy or sell to neutralize the portfolio is exactly $N_F = \frac{\Delta_{\$}}{F_{notional}}$. Continuous maintenance of this zero-delta state isolates the portfolio from first-order directional risks, leaving it exposed primarily to convexity and volatility dynamics.Gamma Scalping and Variance MathematicsA portfolio rendered delta-neutral but holding net-long options possesses a positive Gamma profile. Gamma scalping is the active, continuous recalibration of the delta hedge as the underlying asset fluctuates randomly. The mathematical basis for the profitability of gamma scalping rests on the differential between implied and realized variance. Using a discrete approximation of the Black-Scholes partial differential equation, the expected Profit and Loss (P&L) of a delta-hedged position between time $t$ and $t+\Delta t$ is quantified by the spread between the variance parameterized by the FNO and the actual path variance :$$\text{P\&L} \approx \frac{1}{2} \Gamma S^2 \left( \sigma_{realized}^2 - \sigma_{implied}^2 \right) \Delta t$$If the realized asset volatility over the period exceeds the implied volatility encoded within the derivative's premium, the mechanical process of buying low and selling high to maintain delta neutrality generates cash flows that strictly outpace the deterministic Theta decay (time decay) of the option.Vega Bucketing and Cross-Greek Skew TradingTreating portfolio Vega as a monolithic scalar is a severe abstraction error. The implied volatility surface is highly elastic, and shifts rarely occur uniformly across maturities. Institutional risk engines implement "Vega bucketing," decomposing aggregate Vega exposure into discrete maturity intervals (e.g., $T_1$ through $T_8$). The FNO model captures the dynamics of these buckets through the parameterization of the Rough Heston formulation. Crucially, a perturbation in the Hurst exponent ($H$) predominantly influences the short-dated $T_1$ and $T_2$ Vega buckets, as the power-law singularity characterizing rough volatility asserts maximum influence near $t=0$.Furthermore, traders leverage Vanna ($\frac{\partial \nu}{\partial S}$ or equivalently $\frac{\partial \Delta}{\partial \sigma_{IV}}$) to optimize skew trades. Equity index volatility exhibits a pronounced negative skew, meaning volatility systematically expands during market drawdowns. An option book structured to maintain positive Vanna will witness an endogenous expansion in long Vega exposure exactly as spot prices plummet and volatility spikes. This structural covariance creates an accelerating, protective convexity profile during systemic sell-offs, illustrating why precise evaluation of mixed partial derivatives via PyTorch is critical.Delta-Gamma Hedging Simulation under Rough Heston DynamicsValidating the efficacy of the AD-derived FNO Greeks requires backtesting them within a simulated dynamic hedging environment. To extend a standard delta-hedging simulation to incorporate Gamma hedging, the trader must execute transactions in two distinct instruments: a secondary non-linear derivative (such as a liquid, at-the-money vanilla option) to neutralize the portfolio's aggregate Gamma, followed by a transaction in the linear underlying asset to zero out any residual Delta generated by the option trade.Continuous vs. Discrete Hedging Error FrameworksUnder the idealized assumption of continuous trading without frictions, the Black-Scholes framework postulates perfect, costless replication. In algorithmic practice, trading is discrete, constrained by hardware latency and exchange matching engines, which introduces a structural variance known as the discrete hedging error. According to the limit theorems developed by Hayashi and Mykland (2005), as the number of rebalancing increments $n$ approaches infinity, the normalized discrete hedging error $L_n$ converges in law to a stochastic integral defined by the portfolio's Gamma and the variance of the asset's diffusion :$$\sqrt{n} L_n \xrightarrow{\mathcal{L}} \frac{T}{\sqrt{2}} \int_0^T \Gamma_t S_t^2 \sigma_t^2 dW_t^*$$When transaction costs are incorporated, the hedging error expands dramatically. Leland's (1985) foundational transaction cost model demonstrates that discrete rebalancing subject to proportional bid-ask spreads is mathematically equivalent to operating within a frictionless market evaluated at a heavily modified implied volatility. Leland's adjusted volatility, $\hat{\sigma}$, is defined as:$$\hat{\sigma}^2 = \sigma^2 \left( 1 + \sqrt{\frac{2}{\pi}} \frac{c}{\sigma \sqrt{\Delta t}} \right)$$where $c$ is the round-trip transaction cost rate and $\Delta t$ is the rebalancing interval. In contemporary high-frequency trading simulations, this linear spread model is frequently supplanted by Kyle's (1985) square-root impact model, which posits that market impact slippage scales proportionately with the square root of the trade volume. This non-linear penalty severely degrades the profitability of massive, instantaneous Gamma-hedge rebalances. Consequently, the optimal objective function for the simulation is not solely minimizing the absolute hedging error, but minimizing the Conditional Value at Risk (CVaR) of the terminal P&L distribution. CVaR minimization explicitly heavily penalizes strategies that generate deep left-tail losses due to severe slippage during volatile jumps.Simulating Rough Heston Sample PathsTo construct the simulation environment, one must generate vast arrays of sample paths mirroring the Rough Heston variance process. This process is defined by a fractional Volterra integral equation :$$V_t = V_0 + \frac{1}{\Gamma(H+1/2)} \int_0^t (t-s)^{H-1/2} \kappa (\theta - V_s) ds + \frac{1}{\Gamma(H+1/2)} \int_0^t (t-s)^{H-1/2} \sigma \sqrt{V_s} dW_s$$The presence of the fractional kernel $K(x) = x^{H-1/2}$ creates a singularity at $x=0$. Standard discretization methods, such as the Euler-Maruyama scheme, collapse under this singularity, yielding severely degraded weak convergence rates. To accurately simulate the paths, the industry relies on the Hybrid Scheme introduced by Bennedsen, Lunde, and Pakkanen (2017).The Hybrid Scheme resolves the singularity by bifurcating the convolution integral. Within a tight threshold near $s \to t$ (the singular region), the scheme employs an exact, analytical power-function integration. Outside this threshold (the regular region), the algorithm relies on a highly optimized Riemann-sum step-function approximation. By framing the discrete integration as a matrix convolution, the Hybrid Scheme leverages the Fast Fourier Transform (FFT) to process the entire path with a computational complexity of $\mathcal{O}(N \log N)$.Furthermore, pricing the options along these paths to calculate the mark-to-market P&L requires evaluating the true Rough Heston characteristic function. Unlike the classical Heston model, which relies on a standard Riccati Ordinary Differential Equation (ODE), the rough variant necessitates solving a highly complex fractional Riccati equation :$$D^\alpha h(t, a) = - \frac{1}{2}a(a+i) + i\rho \sigma a h(t,a) + \frac{1}{2}\sigma^2 h(t,a)^2$$Numerical solutions to this equation are generally obtained utilizing the fractional Adams-Bashforth-Moulton predictor-corrector method, which requires iterative calculation of weight arrays ($a_{j, k+1}, b_{j, k+1}$). The following implementation outlines the structure required to evaluate both single-option Greeks and batched parameter sensitivities on CUDA hardware.Pythonimport torch
import torch.func as func
from torch.distributions import Normal

class DifferentiableBlackScholes:
    """
    A purely differentiable implementation of the Black-Scholes framework.
    All scalar parameters must be instantiated as PyTorch tensors to preserve 
    the integrity of the autograd computational graph.
    """
    def __init__(self, r: torch.Tensor, q: torch.Tensor):
        self.r = r
        self.q = q
        # Establish a differentiable normal distribution parameterized at 0, 1
        self.normal = Normal(torch.tensor(0.0), torch.tensor(1.0))

    def d1_d2(self, S, K, tau, sigma):
        """Calculates the d1 and d2 auxiliary probability bounds."""
        d1 = (torch.log(S / K) + (self.r - self.q + 0.5 * sigma**2) * tau) / (sigma * torch.sqrt(tau))
        d2 = d1 - sigma * torch.sqrt(tau)
        return d1, d2

    def call_price(self, S, K, tau, sigma):
        """Computes the exact Black-Scholes Call premium."""
        d1, d2 = self.d1_d2(S, K, tau, sigma)
        return S * torch.exp(-self.q * tau) * self.normal.cdf(d1) - \
               K * torch.exp(-self.r * tau) * self.normal.cdf(d2)

def compute_greeks_single(S, K, tau, r, q, sigma_iv):
    """
    Extracts first and second-order Greeks via nested autograd calls.
    The variables S and sigma_iv must possess requires_grad=True.
    """
    bs = DifferentiableBlackScholes(r, q)
    
    # Forward pass: compute the base analytical price
    C = bs.call_price(S, K, tau, sigma_iv)
    
    # First Order: Delta and Vega extraction
    # create_graph=True ensures the resulting derivative tensors remain attached 
    # to the computational graph for subsequent differentiation.
    delta, vega = torch.autograd.grad(C, (S, sigma_iv), create_graph=True)
    
    # Second Order: Gamma (dDelta/dS), Vomma (dVega/dSigma), Vanna (dVega/dS)
    gamma = torch.autograd.grad(delta, S, retain_graph=True)
    vomma = torch.autograd.grad(vega, sigma_iv, retain_graph=True)
    vanna = torch.autograd.grad(vega, S, retain_graph=True)
    
    return {
        "Price": C.detach(),
        "Delta": delta.detach(),
        "Gamma": gamma.detach(),
        "Vega": vega.detach(),
        "Vomma": vomma.detach(),
        "Vanna": vanna.detach()
    }
To translate the analytical sensitivities back to the fundamental Rough Heston parameters, the system must interact directly with the trained FNO model, executing the chain rule via the jacfwd and vmap algorithms.Pythondef extract_model_sensitivities(fno_model, theta_params, S, K, tau, r, q):
    """
    Maps Option Price gradients back to the 6 Rough Heston parameters 
    by calculating the Jacobian of the FNO surrogate model.
    fno_model: torch.nn.Module mapping (batch, 6) -> (batch, 88)
    theta_params: Tensor of shape (batch_size, 6)
    """
    # 1. Forward inference mapping parameters to the implied volatility matrix
    sigma_iv_surface = fno_model(theta_params) 
    
    # 2. Extract analytic Vega across the entire surface using the differentiable BS framework
    bs = DifferentiableBlackScholes(r, q)
    # sigma_iv_surface must be evaluated with requires_grad=True internally
    C = bs.call_price(S, K, tau, sigma_iv_surface) 
    vega_analytic = torch.autograd.grad(C, sigma_iv_surface, grad_outputs=torch.ones_like(C))
    
    # 3. Compute batched Jacobian of the FNO: d(sigma_iv) / d(theta)
    # jacfwd is preferred as output dimension (88) vastly exceeds input dimension (6)
    fno_functional = func.functional_call
    params_dict = dict(fno_model.named_parameters())
    
    def fno_forward_wrapper(theta):
        return fno_functional(fno_model, params_dict, theta)

    # vmap over the batch dimension, calculating forward-mode Jacobian in parallel
    jacobian_vmap = func.vmap(func.jacfwd(fno_forward_wrapper))(theta_params)
    
    # 4. Chain Rule Application via Einstein Summation: 
    # (batch, 88) @ (batch, 88, 6) -> (batch, 6)
    # The resulting tensor contains the exact sensitivity of the option price 
    # to the underlying Rough Heston parameters.
    model_greeks = torch.einsum('bi,bij->bj', vega_analytic, jacobian_vmap)
    
    return model_greeks
